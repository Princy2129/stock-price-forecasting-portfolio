## 🔍 Overview

This project builds an end-to-end **time series forecasting and portfolio allocation pipeline** for 7 NSE-listed stocks spanning 6 sectors. Three distinct forecasting models are trained, evaluated, and combined into an **optimal ensemble**. The resulting 2-day-ahead price forecasts are fed into a **GARCH(1,1)-based inverse-volatility weighting strategy** to allocate ₹10,00,000 of virtual capital on the StockGro platform.

## 📦 Dataset

- **Source**: [Yahoo Finance via `yfinance`](https://pypi.org/project/yfinance/) — live NSE daily OHLCV
- **Period**: 1 January 2020 – 31 December 2024
- **Frequency**: Daily (business days)
- **Train set**: First ~4.5 years (~1,134 trading days)
- **Test set**: Last 6 months (~126 trading days)
- **Missing value handling**: Forward-fill (limit=3) → Backward-fill (limit=5) → Drop rows only when ALL stocks are NaN

> ⚠️ No data file is committed to this repository. The notebook downloads data live via `yfinance` on every run.

## 🤖 Models Used

### 1. ARIMA (AutoRegressive Integrated Moving Average)
- Applied to **log-returns** (stationary series, d=0)
- Order selected by **AIC-minimising grid search** over p, q ∈ {0, 1, 2, 3}
- Forecasts converted back to price levels via: `P_t = P_{t-1} × exp(r̂_t)`
- **Strength**: interpretable, strong directional accuracy on stable stocks

### 2. LSTM (Long Short-Term Memory)
- 60-day look-back window
- Architecture: `LSTM(64) → Dropout(0.2) → LSTM(32) → Dropout(0.2) → Dense(1)`
- `MinMaxScaler` fitted on **training data only** (no data leakage)
- Trained with EarlyStopping + ReduceLROnPlateau
- **Strength**: captures non-linear long-range price dependencies

### 3. GRU (Gated Recurrent Unit)
- Same architecture and training setup as LSTM
- Fewer parameters than LSTM — faster convergence
- **Strength**: comparable accuracy to LSTM with lower computational cost

### 4. Facebook Prophet
- Multiplicative seasonality mode (appropriate for financial data)
- Weekly + yearly seasonality enabled
- Business-day (`freq='B'`) future dataframe
- 95% confidence intervals plotted
- **Strength**: interpretable trend/seasonality decomposition, robust to missing data

### 5. Optimal Ensemble (ARIMA + LSTM + GRU)
- Grid-search over weight combinations summing to 1.0 (step size 0.1)
- Optimal weights selected by minimising MAPE on the test set per stock
- Prophet excluded from daily ensemble (better suited for weekly+ horizons)

## 💼 Portfolio Strategy

```
Final Weight = 0.5 × GARCH_InvVol_Weight + 0.5 × ForecastReturn_Weight
```

**Step 1 — GARCH(1,1) Inverse-Volatility Weights**

Stocks with higher forecast volatility receive smaller capital allocation:

```
w_i(inv_vol) = (1/σ_i) / Σ(1/σ_j)
```

**Step 2 — Forecast Return Rank Weights (Long-Only)**

Only stocks with positive predicted 2-day return receive weight here:

```
w_i(ret) = max(r̂_i, 0) / Σ max(r̂_j, 0)
```

**Step 3 — Capital Allocation**

```
Shares_i = floor( w_i × 10,00,000 / Price_i )
```

## 🔬 Methodology

```
Raw NSE Data (yfinance)
        │
        ▼
Data Cleaning & Validation
  • ffill / bfill for exchange holidays
  • dropna(how='all') — never drop row unless ALL stocks are NaN
  • Column threshold drop for persistent data gaps
        │
        ▼
Exploratory Data Analysis
  • Rebased price trajectories
  • Return distributions + annualised volatility
  • Rolling 30-day volatility
  • Pearson correlation matrix
        │
        ▼
Stationarity Check (ADF Test)
  • Price levels: I(1) — non-stationary
  • Log-returns: I(0) — stationary → ARIMA on returns
        │
        ├──────────┬──────────┬──────────┐
        ▼          ▼          ▼          ▼
      ARIMA       LSTM       GRU      Prophet
        │          │          │          │
        └──────────┴──────────┘          │
                   │                     │
                   ▼                     │
            Ensemble (ARIMA+LSTM+GRU)   │
            (grid-search optimal weights)│
                   │                     │
                   └─────────────────────┘
                           │
                           ▼
                  GARCH(1,1) Volatility
                  2-day ahead forecast
                           │
                           ▼
              Portfolio Weight Calculation
         (Inverse-Vol + Forecast Return blend)
                           │
                           ▼
              Capital & Share Allocation
```

## ⚠️ Limitations

1. **ARIMA single-pass forecast** — fitted once on training returns, not rolling. Accurate for directional signals, less so for exact prices over 126 steps
2. **LSTM on CPU** — EarlyStopping may halt training early; GPU recommended for full convergence
3. **No transaction costs** — brokerage fees, STT, exchange charges, and slippage are not modelled
4. **2-day horizon** — very short window; highly sensitive to news events, earnings announcements, and macro shocks not captured by historical patterns
5. **yfinance data quality** — adjusted prices may have inconsistencies around corporate actions (splits, dividends, rights issues)
6. **Look-ahead bias** — ensemble weights are optimised on the test set; in production, use a held-out validation set
